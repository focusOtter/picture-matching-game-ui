# MemoryMatch Game - Development Prompt

You are assisting with development of MemoryMatch, a Next.js-based memory card game with Auth0 and Google Drive integration.

## Context
- **Main Tech**: Next.js, React, TypeScript, Tailwind CSS, Radix UI, Zustand, Framer Motion
- **Project Type**: Interactive web game with authentication
- **Current Stage**: Actively developed (v0 bootstrapped project)

## When Asked to Modify Game Logic

1. **Timing & Game States**: Game states are `'countdown' | 'revealing' | 'playing' | 'won' | 'lost'`. Timing constants are in `app/game/page.tsx`.

2. **Card Matching**: Check `app/game/page.tsx` for the matching logic in `handleCardClick`. Matching requires:
   - Exactly 2 cards flipped at a time
   - Image URL comparison to detect matches
   - `isProcessing` state to prevent rapid clicks

3. **Difficulty Settings**: Modify both:
   - `DIFFICULTY_CONFIG` in `lib/game-store.ts` (counts)
   - UI options in `app/page.tsx` (selector)

4. **State Updates**: Always use Zustand store in `lib/game-store.ts` for persistent game state across pages.

## When Asked to Work on UI/Components

1. **Styling**: Use Tailwind classes. Theme colors: `primary`, `secondary`, `accent`, `muted-foreground`, etc.

2. **Animations**: Framer Motion available in most components. Common patterns:
   ```tsx
   <motion.div 
     initial={{ opacity: 0, y: 20 }} 
     animate={{ opacity: 1, y: 0 }}
     transition={{ delay: 0.2 }}
   />
   ```

3. **Forms**: React Hook Form available for form handling in settings.

4. **Responsive**: Use `md:` breakpoints. Test on mobile with `use-mobile` hook.

## When Asked About Authentication/Google Drive

1. **Auth Setup**: Auth0 config in `lib/auth0.ts`. Scopes include Google Drive readonly.

2. **API Routes**: 
   - Route `/api/google-drive` searches folders and lists images
   - Route `/api/google-drive/file/[id]` proxies private file access
   - Both use Auth0 access token for Google OAuth2 connection

3. **Error Handling**: Settings component has error state display. Check `settings-content.tsx` for Google Drive error patterns.

## Before Making Changes

- Check if state should live in Zustand store (multi-page state) vs component state
- Ensure animations don't conflict with existing Framer Motion usage
- Test Google Drive integration requires proper Auth0 + Google OAuth configuration
- Remember 3-minute timer and scoring system when modifying game duration

## Code Patterns to Follow

1. **Client Components**: Add `'use client'` directive for interactive components
2. **State**: Use Zustand for cross-page state, useState for local component state
3. **Types**: Define TypeScript interfaces for complex data (cards, game state, etc.)
4. **Error Handling**: Wrap API calls in try-catch; show toast or error state to user
5. **Accessibility**: Use semantic HTML, ARIA labels, keyboard navigation where applicable

## Common Issues & Solutions

- **Images not loading**: Check `/public/cards/` folder exists and images are there, or verify Google Drive folder name and connection
- **Auth errors**: Verify Auth0 environment variables are set correctly
- **State not persisting**: Use Zustand store instead of component state for multi-page data
- **Styling issues**: Check if Tailwind CSS build includes component files in `tailwind.config.ts`
